Anatomía de un Pago en la Lightning Network

Un Viaje Criptográfico de Alice a Elisa

1. Preparación: El Secreto y su Huella Digital

El proceso comienza con el destinatario final, Elisa. Antes de que se mueva un solo satoshi, Elisa genera la clave criptográfica que desbloqueará todo el flujo: un número secreto (R) y su huella digital o hash (H).

Paso 1: Generación del Hashlock
Elisa (Receptora)1. Genera un secreto (R)
2. Calcula H = SHA256(R)
3. Envía solo el Hash (H)
Alice (Remitente)Recibe H como "condición de pago".

La seguridad de este paso se apoya en la función hash SHA256, que garantiza que es computacionalmente imposible deducir el secreto R a partir de su hash H.

2. Construcción: Encadenando los Contratos HTLC

Con el hash H, Alice construye una ruta de pago segura a través de la red (Alice → Bob → Carol → Elisa) iniciando una cadena de Contratos Hash-Bloqueados por Tiempo (HTLCs) con plazos de tiempo decrecientes.

Paso 2: Cadena de HTLCs con Timelocks Decrecientes
Alice
HTLC (CLTV: 120 bloques)
Bob
HTLC (CLTV: 90 bloques)
Carol
HTLC (CLTV: 60 bloques)
Elisa

Esta estructura de timelocks decrecientes es crucial. Proporciona a cada nodo intermedio un margen de seguridad para reaccionar y reclamar su pago una vez que el secreto se revela, o para recuperar sus fondos si el HTLC expira.

3 y 4. Liquidación: Revelación del Secreto y Cascada Atómica

Una vez que Elisa recibe la oferta de HTLC de Carol, revela el secreto R para reclamar su pago. Este secreto luego fluye hacia atrás a lo largo de la ruta, liquidando cada HTLC en una reacción en cadena atómica: o todo se completa, o todo falla sin pérdida de fondos.

5. Conflicto: ¿Qué Pasa si Alguien Intenta Hacer Trampa?

Si un nodo malicioso (ej. Bob) publica un estado antiguo del canal para robar fondos, el protocolo tiene un mecanismo de defensa robusto: la transacción de penalización.

Escenario de Fraude y Penalización
Bob publica estado antiguoIntenta hacer trampa en la blockchain.
Alice detecta el fraude (gracias al timelock CSV)
Alice usa la Clave de RevocaciónQue Bob le entregó para ese estado.
Resultado Final
¡CASTIGO!Alice reclama el 100% de los fondos del canal.
6. Conflicto: ¿Qué Pasa si un Nodo se Desconecta?

Si un nodo honesto está desconectado, no puede monitorear la blockchain para detectar un intento de fraude. Los Watchtowers resuelven este problema actuando como vigilantes delegados.

Protección con Watchtowers
